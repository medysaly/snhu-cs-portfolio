# SNHU Computer Science Portfolio

## CS-360 Mobile Architect & Programming – App Code Design

This artifact represents the final app code design I developed for CS-360 Mobile Architect & Programming. The goal of the project was to build a functional mobile application that allows users to log in, store persistent data in a SQLite database, display that data in a grid-based interface, and support SMS notifications when permission is granted. The app was designed to address user needs for simple account access, reliable data management, and a clear mobile interface that makes it easy to view and update information.

The main screens and features of the application were designed to support usability and user-centered design. The login functionality allows returning users to authenticate with a username and password while also supporting first-time users who need to create account credentials. After logging in, users can interact with the database through a screen that displays stored records in a grid and supports create, read, update, and delete operations. This structure supports user needs by keeping navigation straightforward and making key actions easy to access from the main workflow.

When developing the application, I approached the code by building one piece of functionality at a time and testing each feature as I went. I focused on keeping classes concise, using clear naming conventions, and writing code that was readable and maintainable. I also used consistent logic patterns for handling login validation, database operations, and screen interactions. These strategies helped me stay organized during development and are approaches I will continue using in future software projects.

Testing was a major part of the development process because the app needed to function correctly across multiple features. I used the Android Emulator to test login behavior, account creation, database persistence, CRUD operations, and the SMS permission workflow. Testing helped me identify issues such as incorrect input handling, navigation errors, and logic problems in how information was stored or displayed. This process reinforced how important testing is for creating a stable and user-friendly mobile application.

One place where I had to innovate was designing the application so it would continue to function correctly whether the user granted SMS permissions or denied them. This required me to think carefully about how to separate optional notification behavior from the core app functionality. By doing that, I was able to make sure the database and app workflow still worked normally even if messaging permissions were not allowed, which improved the overall reliability of the design.

The part of the project that best demonstrates my knowledge and skills is the integration of user login, persistent SQLite storage, and CRUD functionality in one application flow. This component shows my ability to connect the user interface to underlying application logic and create a mobile app that is both functional and structured around real user tasks. It reflects my growth in Android development and my ability to build a complete application rather than only designing static screens.
